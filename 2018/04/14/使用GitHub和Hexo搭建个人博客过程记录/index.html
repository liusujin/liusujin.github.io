使用GitHub和Hexo搭建个人博客过程记录

发表于 2018-04-14 | 分类于 学习
字数统计: 1,550 字 | 阅读时长 ≈ 6 分钟

搭建个人博客是自己源于自己大学时就萌生的一个想法，但是当时苦于找不到好的入门资料及教程而被顺势搁置。两天前在逛GitHub的时候看见了使用GitHub Page搭建博客的主题，心里又重拾了建个博客写点什么的想法。

在参考网上的教程使用GitHub Page和Hexo 建立博客的过程中也出现了很多问题，所以觉得有必要将这个过程记录下来，给后来者和自己做以参考,当然这也是一个折腾和学习的过程。

  • 推荐按照Hexo官方安装文档操作

安装前电脑环境相关：

  • windows10 64位
  • 已安装GitHub Desktop
  • 已安装vs code编辑器
  • 其他常用运行库

1.注册GitHub账号及建立仓库

注意事项：

仓库名称为username.github.io，其中username为我们的注册用户名,这个就是我们的博客访问域名

2.安装Node.js

  • 下载地址：http://nodejs.cn/
  • 参考安装教程：http://www.runoob.com/nodejs/nodejs-install-setup.html
  • 安装完成后打开cmd，输入命令node -v后回车，出现版本说明则安装成功

3.安装Git及Hexo

  • 参考http://msysgit.github.com/
  • 安装完成后打开cmd（建议以管理员权限运行），输入命令git --version后回车，出现版本说明则安装成功
  • 找到桌面的Git Bash.lnk双击进入
  • 分别输入我们的GitHub用户名和验证邮箱
  • $ git config –global user.name “username”
  • $ git config –global user.email “xxx”
  • cmd中输入npm install -g hexo-cli
  • 耐心等待安装完成（如果出现WARN错误可以忽略）

4.初始化博客文件及设置

  • 选择一个文件夹，随后将在这个文件夹生成博客文件夹
  • 以管理员权限运行cmd并进到当前目录（或者在当前目录Shift+鼠标右键后选择在此处打开Powershshell窗口）
  • 执行hexo init blog（同样出现WARN错误可以忽略）
  • 新建blog文件夹中部分文件夹及文件说明
    • _config.yml，网站的配置信息，您可以在此配置大部分的参数
    • source，资源文件夹是存放用户资源的地方。除 posts 文件夹之外，开头命名为 (下划线)的文件/文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。
    • themes，主题 文件夹。Hexo 会根据主题来生成静态页面
  • 出现提示Start blogging with Hexo则说明初始化成功
  • Hexo在路径source/_post/下自带一篇Hello World.md博客
  • 命令行进入blog目录下

    常用操作指令
    hexo n “文章名称” == hexo new “文章名称” # 新建文章
    hexo clean # 清除已生成的预览文件和缓存
    hexo g == hexo generate # 生成
    hexo s == hexo server # 启动服务预览,默认情况下，访问网址为： http://localhost:4000/
    hexo d == hexo deploy # 部署，上传到GitHub（既可以通过username.github.io进行访问）

  • 输入hexo g，生存静态文件
  • 输入hexo s，启动服务器
  • 已管理员权限打开cmd，输入ssh-keygen -t rsa -C “Github的注册邮箱地址”后一路回车
  • 使用vs code打开生成的id_rsa.pub（id_rsa.pub的作用和获取id_rsa.pub）后Ctrl + A复制所有内容
  • 登陆GitHub后点击右上角用户头像Settings->SSH and GPG keys->New SSH key->Title:blog->Key:刚才复制的内容->Add SSH key

5.博客网站配置信息

  • 使用vs code打开blog文件夹下的_config.yml文件修改参数
  • 每个参数的：后都要加一个空格
  • 博客基本信息修改

    -title: 博客名称
    subtitle: 子标题
    description: 描述
    author: 作者
    language: 语言
    timezone: 时区

  • 部署服务器的相关配置信息，只需要修改repo部分，username设置为你的username

    deploy:
    type: git
    repo: https://github.com/username/username.github.io.git
    branch: master

  • 在cmd中进到blog目录下执行命令
    • $ hexo clean
    • $ hexo g
    • $ hexo s
  • 打开localhost:4000/后检查,确认无误后使用hexo deploy部署到服务器上
    • 出现github登录界面，正常填写登陆就行

6.主题修改

  • 官方hexo主题大全

下面以我个人最喜欢的NexT主题为例说明下主题的更换步骤：

  • 到主题相应的GitHub链接中下载Source code (zip)
  • 将下载文件解压到目录blog/themes/
  • 将blog/_config.yml中的theme属性设置修改为theme: next
  • 再进行一次hexo -clean,hexo -g,hexo -s预览修改后的效果
  • 确认无误后hexo -d进行部署

7.部分配置修改

7.1.修改首页菜单栏

  • 在路径blog/themes/next/_config.yml对menu下的属性进行修改即可实现
  • 以# categories: /categories/ || th为例，修改为categories: /categories/ || th后菜单栏将会多出分类子选项,但是此时点击是没有反应的，需要执行hexo new page categories
  • 在blog/source/下会新生成一个新的文件夹categories ，在该文件夹下会有一个index.md文件,对其进行修改

    -–
    title: categories
    date: 2018-04-14 15:36:29
    type: categories
    -–

-此时分类按钮可以正常点击

7.2修改Hexo主题

  • Hexo具有四种主题，默认为Muse(可以按照自己的喜好进行修改，个人偏好Mist)

    # Schemes
    # scheme: Muse
    scheme: Mist
    # scheme: Pisces
    # scheme: Gemini

7.3设置预览摘要

  • 在路径blog/themes/next/_config.yml对auto_excerpt下的属性进行修改

    auto_excerpt:
    enable: true # 将原有的false改为true
    length: 300 # 设置预览的字数

8.总结

整个过程就是一个折腾和学习的过程，在这个折腾的过程中学习了Git的使用，npm包管理器的使用，SSH相关等知识。

9.问题备份

9.1使用hexo d部署时出错，提示ERROR Deployer not found: git,解决方法：

执行命令npm install --save hexo-deployer-git后再hexo -d即可

9.2markdown解析不了标题(#)

#后面要加空格，不能直接跟标题文字

9.2文章右侧目录索引重复

在路径blog/themes/next/_config.yml对toc下的属性number修改为false
